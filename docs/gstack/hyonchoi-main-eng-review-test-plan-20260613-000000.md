## Key Interactions to Verify

- **Mid-flight re-pick blocked (A1):** at T=0 a phase starts and writes
  `.hermes/phase_started/TODO-N.json`; at T=5 a new tick fires and builds
  SelectionContext ‚Äî `in_flight` MUST contain `TODO-N` even though
  `ready_for_review` has not been written yet.
- **Outcome feedback (A3):** after a phase transitions to
  `merge_status: failed` at Phase 4, the decision JSON for that tick is
  updated with `outcome: "failed_at_phase_4"` via O(1) tick_id path lookup;
  the NEXT tick's `recent_decisions` carries that outcome to the agent.
- **Prompt SHA pin (A2):** with `expected_prompt_sha` unset, decisions
  record the actual SHA and never fail; with `expected_prompt_sha=X` and
  live prompt SHA=Y, `run_selection()` returns `picked=None` with
  rationale containing both SHAs, counts as no-progress, never calls the
  Anthropic API.
- **Per-tick config reload:** edit `.hermes/config.toml` mid-run, next
  `pipeline-tick` invocation sees the new values (no caching, no daemon).
- **Phase 9 merge (REGRESSION):** typed-confirm path retains exact prior
  behavior after `phases.py` extraction from `watcher.py`. Pin behavior
  with a regression test BEFORE the extraction lands.

## Edge Cases

- Anthropic API errors: 429 / 401 / 5xx (retry once) / timeout ‚Äî all
  produce `picked=None` with distinct rationale strings; circuit breaker
  counts each as no-progress.
- Parse failures: invalid JSON / missing `picked` / `picked` references a
  TODO not in `candidates_considered` ‚Üí `picked=None`.
- Stale `phase_started.json`: marker older than `max_phase_timeout` swept
  on next read in `decision/context.build_in_flight()`. Guards against
  Claude process crashes leaving orphan markers.
- Decision archive rotation: hot dir at exactly 50 = noop, at 51 = oldest
  moved to `archive/`.
- `recent_decisions` carrying mixed legacy (no `outcome` field) and new
  (with outcome) ‚Äî agent receives `outcome: null` for legacy, agent.py
  must serialize None cleanly.
- Concurrent state.py outcome write while store.py rotation is running ‚Äî
  atomic file move semantics on POSIX; verify on the target filesystem.

## Critical Paths (end-to-end, must work)

- **Shadow-mode tick:** cron ‚Üí `pipeline-tick` ‚Üí `run_selection()` ‚Üí JSON
  written to `.hermes/decisions/<tick_id>.json` ‚Üí STOP (auto_execute=false).
  Human reads decision, runs `hermes run pipeline-phase --todo X --tick-id Y`
  manually.
- **Live-mode happy path:** cron ‚Üí decision ‚Üí `pipeline-phase` async ‚Üí
  phase writes `phase_started/X.json` at start ‚Üí Claude Code runs ‚Üí phase
  writes `ready_for_review` and deletes `phase_started/X.json` on success.
- **Live-mode concurrent tick:** at T=5 with phase still running, new
  tick fires, agent sees `in_flight=[TODO-N]`, picks something else or
  returns `picked=None`.
- **Live-mode failure loop:** phase fails at Phase 4 ‚Üí outcome written ‚Üí
  next tick agent sees the failure in `recent_decisions` ‚Üí picks a
  different TODO (the A3 payoff).
- **Circuit breaker:** force 3 consecutive `picked=None` ticks ‚Üí cron
  interval bumps to 30 min ‚Üí one Slack alert via `hermes chan message`
  ‚Üí next successful pick resets counters ‚Üí cron interval restored.
- **Phase 9 merge gate (REGRESSION):** typed-confirm flow unchanged.

## Eval Suite (T1 ‚Äî minimal, CI non-blocking until bake week ends)

`tests/eval/selection/` with 8-12 fixtures:
- `drift_strict.md` ‚Äî fully formatted TODOS.md, agent must pick highest P1.
- `drift_mild.md` ‚Äî half-filled metadata, freeform notes interspersed.
- `drift_heavy.md` ‚Äî sections reordered, missing Priority fields.
- `inflight_respected.md` ‚Äî TODO-2 is in_flight; agent must not pick it.
- `outcome_aware.md` ‚Äî recent_decisions shows TODO-3 failed twice at
  Phase 4; agent must pick something else (or justify why TODO-3 is
  still right despite the history).
- `priority_conflict.md` ‚Äî two P1s, one with `Depends on: TODO-X`;
  agent must skip the dependent.
- `sha_mismatch.md` ‚Äî config pins `expected_prompt_sha=X`; live prompt
  computes Y; agent must not be called.
- `empty_todos.md` ‚Äî no candidates ‚Üí `picked=None`, rationale=
  "no candidates available".

Each fixture: `input/TODOS.md` + `input/recent_decisions.json` +
`input/config.toml` + `expected.json` (either an exact `picked` or a
predicate the picked TODO must satisfy).

Runner: `pytest tests/eval/selection -m eval`. CI workflow runs it on
every commit touching `decision/agent.py` or `.hermes/prompts/`.
Non-blocking until bake week ends; results posted as a PR comment.
Flip to blocking after the live-mode bake by setting
`cancel_in_progress: false` and removing `continue-on-error: true` from
the GitHub Actions job.
