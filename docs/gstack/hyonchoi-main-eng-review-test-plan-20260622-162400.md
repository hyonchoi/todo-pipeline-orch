## Affected Commands
- `pipeline-watch tick` — now accepts `--all` or no args (scan all projects). `tick <project>` preserved for manual debug.
- `pipeline-watch status` — verify it works with per-project state dirs
- `pipeline-watch kill` — verify it works with per-project state dirs

## Key Interactions to Verify
- Running `pipeline-watch tick` with no projects in projects_dir — clean exit, no error
- Running `pipeline-watch tick` with one project that has project.toml — selection runs, kanban updates
- Running `pipeline-watch tick` with multiple projects — all active projects processed
- Running `pipeline-watch tick` with one project that has malformed TODOS.md — that project skipped, others proceed
- Running `pipeline-watch tick` with one project that has malformed project.toml — that project skipped with ERROR log, others proceed
- Running `pipeline-watch tick` with a project that has no project.toml — project skipped (opt-in required)
- Running `pipeline-watch tick <project>` — preserved for manual debugging
- Running `pipeline-watch tick` with Hermes unresponsive — project times out after 120s, others continue
- State migration on first run with single project — global state moves to per-project dir
- State migration on first run with multiple projects — migration skipped, warning logged

## Edge Cases
- Empty projects_dir — scan returns empty list
- Project without TODOS.md — skipped by discovery
- Project with enabled=false in project.toml — skipped
- Invalid project slug (starts with dash, contains ..) — skipped with warning
- Disk full during state migration — global state preserved (copy-then-delete)
- Channel resolution: project.toml > env var > #alert — test all three priorities
- Concurrent tick during migration — quiescence check prevents split state

## Critical Paths
- Full scan: tick → discover projects → for each: prior-tick check → selection → kanban registration
- Error isolation: one project fails → others continue → exit code reflects partial failure
- Migration: single project → global state migrates → subsequent scans use per-project state
