## Test Diagram — every new thing this plan introduces

```
NEW CODEPATHS:
  - runner/dispatcher gate check (BLOCKED/READY/RUNNING/FAILED/UNKNOWN branches)
  - approve-plan CLI: --approve / --reject / --override / --reason
  - check_gate_status() pure read
  - stub decision-sheet generator (markdown → JSON)
  - DecisionSheet load-time validation
  - rejection_count increment + manual_queue transition

NEW DATA FLOWS:
  - Autoplan markdown plan → stub generator → decision sheet JSON
  - approve-plan → decision sheet answer fill → Kanban task complete
  - reject → rejection sidecar → Kanban failed → TODOS.md annotation

NEW INTEGRATIONS / EXTERNAL CALLS:
  - hermes kanban complete/create/list (gate task lifecycle)
  - approve_lock (fcntl flock)

NEW ERROR/RESCUE PATHS:
  - missing decision sheet / corrupt JSON / schema mismatch
  - lock contention ("gate already being processed")
  - unknown tick_id / gate task not found (UNKNOWN)
  - override: unknown q_id / duplicate / invalid label
```

## 2am-Friday tests (the ones a hostile QA writes)

- Approval crashes AFTER decision-sheet JSON write but BEFORE Kanban complete → re-run is idempotent, no double-approve.
- Stale/missing decision sheet while gate is BLOCKED → clear operator message, no deadlock.
- Two ticks for same TODO race the gate → lock serializes, no split sheet.
- SIGKILL during `bump_in_pr` (reused by ship path) → git worktree isolation prevents operator-tree corruption (ENG-M5).
