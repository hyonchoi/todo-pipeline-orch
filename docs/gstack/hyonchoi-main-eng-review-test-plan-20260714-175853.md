## Key Interactions to Verify
- `hermes-pipeline test --fixture happy-path` boots a mock project and runs the full pipeline
- `--loop` re-creates fixture from scratch and diffs against previous report
- `--keep` preserves temp dir for inspection
- Convergence detector halts after 3+ consecutive same-class failures
- Dependency-aware skip prevents running phases whose prerequisites failed

## Edge Cases
- First `--loop` run with no prior report file
- Empty JSONL event log (pipeline fails before any phase starts)
- All 9 phases fail — report still generates
- Hermes CLI not installed — preflight fails fast
- Phase times out — records `timeout`, continues to next phase
- Callback exception — doesn't interrupt pipeline

## Critical Paths
- End-to-end: fixture bootstrap → PipelineRunner invocation → JSONL event log → report generation → loop diff
- Config isolation: HOME override + pinned model prevents touching real pipeline state
- continue_on_failure=False (default) — existing callers unchanged
