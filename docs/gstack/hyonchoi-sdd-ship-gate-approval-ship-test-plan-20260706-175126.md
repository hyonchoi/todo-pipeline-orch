## Key Interactions to Verify
- `hermes approve` exit codes: 0 success, 2 unknown project, 3 refused (guards)
- approve_ship idempotency: re-running after merge is a no-op
- Pre-merge guards: dirty tree refuse, stale SHA refuse, double --force bypass
- CI-green gate: approve blocks until CI checks pass (squash merge on match-head)
- Sidecar atomic write/read, corrupt-file skip, dedup on maybe_ship_ready

## Edge Cases
- No sidecar / no gate task → approve refuses cleanly
- No CI checks configured → warning, proceeds
- get_todo_kanban_tasks: CLI failure → empty; exception handler (timeout/JSON) [GAP]
- bump_in_pr restores original branch on failure

## Critical Paths
- Full approve flow: gate detected → guards pass → bump_in_pr → CI green → squash merge → gate task completed
