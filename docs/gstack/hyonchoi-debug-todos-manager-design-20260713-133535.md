## Problem Statement

Users frequently need to see what TODOs exist without running a full audit. The current `--audit` subcommand is a compliance check (missing fields, invalid deps, ID gaps) — not a navigation tool. There's no lightweight way to answer "what TODOs do I have?" without parsing a verbose audit report.

## What Makes This Useful

A one-command summary of active TODOs: ID, status, title, one-line summary. Optionally includes archived entries with `--all`. Reduces context-switching overhead — the user doesn't need to open TODOS.md and scan 17 entries manually.

## Constraints

- **Skill-only change.** The todos-manager is a prompt-based skill (SKILL.md), not a Python CLI. `--list` means "when the user invokes `todos-manager --list`", the skill parses TODOS.md and displays output.
- **Output is plain text.** Chat context, markdown-rendered. No browser artifact, no interactive component.
- **Scope is `--list` and `--all`.** No filtering by status, priority, search, or sort. Those are future extensions.
- **Read-only.** The command parses and displays; it modifies no files.

## Premises

1. This is a skill-only change (SKILL.md section addition), not a Python CLI command.
2. Output is plain-text markdown rendered in chat.
3. The `--all` flag is the only modifier — show archived entries in addition to active.
4. The user agreed to all three premises and fast-tracked to design.

## Approaches Considered

### Approach A: Inline skill workflow (CHOSEN)

Add a `--list` workflow section to SKILL.md. When invoked, read TODOS.md (and optionally TODOS-archive.md), regex-parse all TODO-<n> header lines, extract status/title/summary, and display as a formatted markdown list.

- Effort: S (~30 lines in SKILL.md)
- Risk: Low
- Reads from existing files; no new code
- Follows existing skill pattern (--audit already does similar scanning)

### Approach B: Skill workflow + Python helper script

Same as A, but add a small Python script that parses TODOS.md with regex and outputs structured output. The skill invokes it via Bash for deterministic parsing.

- Effort: S (two files)
- Risk: Low
- Over-engineering for a read-only list command

### Approach C: Extend --audit to include listing

Add listing output to the top of the audit report.

- Blurs concerns — audit is compliance, list is navigation
- Misses the stated goal of reducing context-switching overhead

## Recommended Approach

**Approach A: Inline skill workflow.** Smallest diff that ships the goal. One new section in SKILL.md. If filtering, search, or CLI integration becomes valuable later, Approach B is the natural evolution.

### Workflow Specification

Add to SKILL.md under `## Workflow`:

```
### `--list`: List active TODO entries

1. **Validate context:** Does TODOS.md exist? If not, print "TODOS.md not found. Run `todos-manager --init` first." and exit.
2. **Scan TODOS.md** for all lines matching `- (\[[ →x~]\])` containing `TODO-(\d+)`.
3. **If no entries found in TODOS.md:**
   - If `--all` was passed: skip the active table (do not exit) and continue to step 6 to show archived entries.
   - If `--all` was NOT passed: print "No active TODOs found." and exit.
4. **For each match**, extract:
   - Status marker: `[ ]` → Pending, `[→]` → In Progress, `[x]` → Done, `[~]` → On Hold
   - ID: `TODO-<n>`
   - Title: text between `TODO-<n>: ` and the closing `**` bold delimiter (strip `**` markup)
   - Summary: text after ` — ` on the header line. If ` — ` is not present, display `[no summary]`.
   - If any field cannot be extracted from a matching line, display `[not set]` in the corresponding column.
5. **Display output** as a formatted markdown table (entries sorted by ID ascending):
   ```
   ### Active TODOs

   | ID | Status | Title | Summary |
   |----|--------|-------|---------|
   | TODO-1 | Pending | Example title | One-line summary |
   ```
6. **If `--all` flag is present**, also scan TODOS-archive.md (if exists):
   - Parse with the same rules (steps 2-4)
   - Display as a separate table section labeled "Archived TODOs" below the active table
   - If TODOS-archive.md does not exist or contains no entries, skip the archived section silently
7. **Print summary line:**
   - Without `--all`: "Showing N active entries."
   - With `--all`: "Showing N active entries. M archived entries."

Report only — no files modified.
```

## Open Questions

- **Sorting:** Should entries be sorted by ID (ascending) or by status (pending first)? Default to ID ascending since that matches document order.
- **Truncation:** If a title or summary is very long, should it be truncated? For now, show full text. If a user has 100+ TODOs with long summaries, that's a future optimization.

## Success Criteria

- User runs `todos-manager --list` and sees a formatted table of all active TODOs in TODOS.md.
- User runs `todos-manager --list --all` and sees active + archived entries.
- No files are modified by the command.
- The skill section is ~30 lines in SKILL.md.
